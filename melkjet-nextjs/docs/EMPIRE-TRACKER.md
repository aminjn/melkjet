# 🏛 ردیابِ رسمیِ Empire Bible — منبعِ حقیقتِ پیاده‌سازی

> **این فایل حافظهٔ پروژه است.** هر سشنِ جدیدِ Claude اول این فایل را می‌خواند.
> با هر سندِ جدید یا فازِ جدید، همین فایل به‌روز و کامیت می‌شود. چت حافظه نیست — این فایل هست.

## 📜 قوانینِ نقض‌نشدنی (از خودِ سندها + دستورهای صریحِ امین)

1. **هیچ دادهٔ فیک، هرگز.** همهٔ اعداد از بازارِ واقعیِ ملک‌جت (آگهی‌ها/رویدادها/آمار). اگر داده نیست → صادقانه خالی/پیام، نه عددِ ساختگی.
2. دستیارِ بازی **«ملک‌جت»** است — نامِ «نورا/آرمان/سارا» ممنوع (دستور صریح کاربر).
3. در UI کاربر هرگز واژهٔ **«بازی»** نمی‌آید → «مسیرِ رشد / امپراتوری / سفرِ مالی» (Branding Rule سند).
4. هر عددِ موتور باید در سوپرادمین قابل‌تنظیم باشد و **واقعاً** موتور زنده را عوض کند (نه فیلد تزئینی).
5. هیچ Pay-to-Win؛ پرداخت فقط برای سرعت/تحلیل/ظاهر.
6. «هیچ پولی از هیچ ساخته نمی‌شود» — هر تولید/مصرف پول منبع و مقصد دارد (مالیات → خزانه، بدهی از netWorth کم می‌شود).
7. مکانیک‌های شانسی (صندوقچه/مذاکره/کوئست) **قطعی از هش** — ضدسوءاستفاده، تست‌پذیر.
8. امپراتوری **منوی مستقلِ خودش** را در سوپرادمین دارد (گروه «امپراتوری (بازی)») — هیچ‌چیز داخل REOS قاطی نشود.
9. هر فاز: تست یونیت + تستِ PG واقعی + tsc + build + کامیت + push. دیپلوی همیشه دستِ کاربر: `sudo scripts/deploy.sh` + پاک‌کردن کش آروان.
10. `Date.now()` بازی‌ساز نیست — snapshot/دوره‌ها بر پایهٔ dayNumber؛ ورودی‌های عددی فارسی/جداکننده‌دار همیشه با `digitsOf` پاک می‌شوند.

### سند ۱۲ (جلد۷۳–۷۸ + سه تصمیمِ معماریِ نهایی)

⏳ **باقی از این سند (با تریگر):** رقبای شخصیت‌دار و فرصت‌های زمانی بازار (بخش ۵) → NPC Civilization؛ تیم‌سازی سریع/حرفه‌ای/اقتصادی + فرهنگِ سازمانی + بحرانِ سه‌گزینه‌ای شرکت (بخش ۳) → وقتی چندپروژگی و تیمِ بزرگ رایج شد؛ DNA زمین/شایعه‌ها («شنیده می‌شود مترو…»)/زمین‌های داستانی (بخش ۶) → نیازمندِ لایهٔ خبر/شایعهٔ راستی‌آزمایی‌شده (Event Engine v2)؛ گسترش به شهرِ جدید/تغییرِ مسیرِ فعالیت (بخش ۷) → وقتی چند بازیکنِ سطح‌بالای واقعی داریم؛ بازارِ هدف (خانواده/لوکس/دانشجو…) و فازبندیِ سرمایه‌گذاری (بخش ۸) → نیازمندِ Customer Engine.

### سند ۱۴ (GDD فصل ۴ بخش‌های ۹–۱۶ + پیوتِ بزرگِ «مدل B»)
📌 **پیوتِ تعیین‌کننده (لازم‌الاجرا برای همهٔ فازهای بعد):** MelkJet **بازی** است نه شبیه‌ساز — «Tycoon + Strategy + Collection + Live Economy» (مدل B: مونوپولی‌گو/کلش‌وار، نه Football Manager). آزمونِ هر قابلیت: «آیا کاربر می‌گوید *فقط ۵ دقیقهٔ دیگر*؟» + هر قابلیت باید تصمیم/ریسک/فرصتِ جدید بسازد. ترتیبِ طراحی: **Hook → Daily Gameplay → Meta Progression → پشت‌صحنه**. واژه‌های ERP/مدیرعامل/حسابداری/فرآیندِ اداری ممنوع مگر مستقیماً لذت بسازند. Big Data محصولِ طبیعیِ رفتارِ بازیکن است، نه چیزی که حس شود.
✅ **فاز ۱۶ ساخته شد:** Hook فرصت‌های محدودِ روزانه (بخش ۱۰ + «۹ شب چه می‌بیند»)، حافظهٔ مذاکره (بخش ۹)، اعتبارِ اثرگذار روی مذاکره/بانک (بخش ۱۵ §۶). مذاکرهٔ چندگزینه‌ای (قبول/کمتر/اقساط/مشارکت/ترک) نسخهٔ فعلی: مذاکرهٔ قطعی + شخصیتِ مالک + حافظه — گزینه‌های اقساط/مشارکت با NPC/قرارداد می‌آیند.
⏳ **باقی از این سند (با تریگر):** رقبای شخصیت‌دار که پروژه می‌سازند و روی قیمت اثر می‌گذارند + رقابت بر سرِ زمین/مشتری/نیرو + M&A (بخش ۱۱) → NPC Civilization؛ بازاریابی/کمپین (بخش ۱۲) و مشتریِ شخصیت‌دار/رضایت/وفاداری (بخش ۱۳) → Customer Engine (بدونِ شبیه‌سازیِ تقاضا، اثرِ صادقانه ندارند)؛ R&D/مسیرِ توسعه/مزیتِ تخصصی (بخش ۱۴) → همان صفِ Prestige/Skill Tree؛ شراکت/سرمایه‌گذار (بخش ۱۶) → خودِ سند آن را «تلهٔ شبیه‌ساز» خواند — فقط نسخهٔ Gamified (مثلِ مشارکتِ جمعیِ موجود)؛ اعتبارِ محلی/تخصصی (بخش ۱۵ §۸/۹) → وقتی چندشهر فعال شد؛ مذاکره با دولت/طبقهٔ اضافه (بخش ۹ §۵) → Event Engine v2.

- «Every Entity Has a Location. Every Action Happens in a Workspace.» — هر موجودیت مختصات دارد (آگهی‌های واقعی دارند؛ موجودیت‌های جدید هم باید داشته باشند).

⏳ **جلد۴۲ (صنعت ساخت/WBS/CPM)** — ERP ساختِ واقعی؛ پایه‌اش پنل سازنده (builder-store با پروژه/واحد/مایلستون واقعی) موجود است؛ نسخهٔ گیم‌پلی منتظر NPC/پیمانکار.

⏳ **جلد۴۸ (Workspace OS)** — فقط اعلام شده؛ محتوایش هنوز نیامده.
📌 **Block 8 (World Experience — نقشهٔ جهانی/3D/Simulation Mode/Time Machine)**: خودِ سند می‌گوید «جای آن الان نیست» و در Block مستقل ثبتش کرده — هر وقت جلدهایش آمد، Phase B شروع می‌شود.

### سند ۹ (جلد۴۸–۵۳)
**قانونِ سه‌گانهٔ جدیدِ سند (از جلد ۵۲):** هر اتفاقِ مهم سه خروجی دارد — Simulation (اثر روی موتور) + News (روایت) + Memory (ثبتِ همیشگی). وضعیتِ فعلی: Memory = لاگِ رویدادهای REOS + تایم‌لاین/claims (پاک نمی‌شوند)؛ News = روزنامهٔ فاز ۱۰؛ Simulation = رویدادمحوریِ REOS.

✅ **جلد۴۹ (Engagement/Retention) — فاز ۱۰ + قبلی‌ها:** کوئست روزانه/هفتگی، استریک، صندوقچه، نشان‌ها/مخفی، کلکسیون، لیگ‌ها، Comeback (فازهای ۲–۸) + حالا: کنسول Engagement ادمین (فصل ۱۹/۲۰: DAU/WAU/MAU/D1/D7/D30/churn) و Mastery (فصل ۵). ⏳ فصلی/Battle Pass/رویداد زنده → Season Engine (صف #۲)؛ Retention AI پیشنهاددهنده → فعلاً لیستِ churn + هدیهٔ دستی ادمین.
✅ **جلد۵۲ (روایت/Event Bus) — فاز ۱۰ نسخهٔ ۱:** روزنامهٔ ملک‌جت از رویدادهای واقعی (بدونِ اسکریپت). ⏳ روزنامهٔ per-city، شایعه، افکار عمومی، سنجشِ NPC → منتظر NPC.
✅ **جلد۵۱ (زمان/تاریخ) — جزئی:** Memory Layer (رویدادها/تایم‌لاین حذف‌نشدنی)، آرشیو رکوردها (فصل ۹)، ردِ فعالیت تاریخ‌دار. ⏳ تاریخچهٔ قیمتِ هر آگهی (نیاز به snapshot دوره‌ای — تریگر: تصمیم برای جدول price_history)، نقشهٔ تاریخی/نسل‌ها → NPC/Block 8.
⏳ **جلد۴۸ (Workspace OS):** روحش تا حدی هست (پنل‌ها + منوی مستقل + AI copilot در پنل‌ها)؛ نسخهٔ کامل (Command Palette، پنجره‌های شناور، Multi-Workspace، همکاری Realtime) = بازطراحی کامل UI → Block 8/Phase B با تأیید کاربر.
⏳ **جلد۵۰ (سازمان/گیلد چندبازیکنه):** «واحدِ بازی = سازمان» — نیازمندِ جمعیتِ همزمانِ بازیکنانِ واقعی؛ ساختنش الان = گیلدهای خالی. تریگر: چند ده بازیکنِ فعالِ واقعی. زیرساختِ آماده: referral دوطرفه، لیدربرد، trust.
⏳ **جلد۵۳ (دنیای خودگردان):** «The World Never Waits» — همین حالا جزئی برقرار است (هزینهٔ مالکیت/بهرهٔ وام/درآمد اجاره/سود صندوق در غیابِ کاربر جلو می‌روند + Comeback). نسخهٔ کامل (شرکت‌های خودمختار) → NPC.

### سند ۱۰ (جلد۵۳–۵۷ + سه بازنگری معماری)
**قوانینِ معماریِ رسمیِ سند (تا امروز ۴ قانون):** ۱) Every Entity Has a Location ۲) Everything Is Extendable ۳) Never Show Data If You Can Show Experience ۴) Every Object Must Be Inspectable. + اصلِ World-first (دنیا حولِ بازیکن نمی‌چرخد) + از جلد ۵۴ هر جلد لایهٔ Production Blueprint دارد.

✅ **جلد۵۶ (UX/Motion/Delight) — فاز ۱۱ نسخهٔ ۱:** motion پلکانی، میکرواینترکشن، CountUp پول، جشنِ موفقیت، خطِ آسمان (قانون ۳ روی دارایی‌ها). ⏳ صدا/هپتیک، دوربینِ سینمایی، Scene-محوری کامل، حالتِ 3D/World Mode → Block 8 (نیازمند موتور نقشه/3D و تصمیمِ کاربر).
✅ **جلد۵۴ (انسان در برابر AI) — روحش برقرار:** AI همه‌جا مشاور است نه تصمیم‌گیر (تحلیل/Twin/پیشنهاد با اطمینان٪)، تصمیمِ نهایی با کاربر؛ حالا توضیح‌پذیری (چیپ‌های چرایی) هم اضافه شد؛ مانیتورِ AI ادمین (ai-cost/مدل‌ها) از قبل. ⏳ تیمِ چند-AI شخصیت‌دار (AI CFO/Lawyer/…) → نیازمند تعریفِ agentهای جدید در AI Studio.
⏳ **جلد۵۳ (دنیای خودگردان کامل):** NPC-قفل؛ برقرارِ فعلی: اقتصادِ بازیکن در غیابش جلو می‌رود + cron نامه‌ها. «AI شرکت/دولت/شهر» → تمدن NPC.
⏳ **جلد۵۵ (Platform SDK/افزونه/Marketplace):** بازطراحیِ Core در مقیاسِ تیمی — روی این کدبیس تک‌مخزنه فعلاً واقع‌بینانه نیست؛ روحِ فعلی: flags، config زنده، منوهای ماژولار. تریگر: تیمِ چندنفره/تصمیمِ کاربر.
⏳ **جلد۵۷ (CityOS/ترافیک/زیرساخت/Digital Twin شهر):** نیازمند GIS کامل (جلد۴۷) + NPC + Block 8؛ «Inspectable» تا حدِ لینکِ آگهیِ واقعی و پروندهٔ بازیکن برقرار است.

### سند ۱۱ (جلد۵۸–۶۵ + قفلِ نقشه‌راه روی گیم‌پلی)
**قانونِ ۵ سند:** از جلد ۶۱ هر جلد باید به ۴ سؤال جواب دهد (بازیکن امروز چه می‌کند/چه مأموریتی جلو می‌رود/چه Unlock می‌شود/چطور اعتیادآورتر). **قولِ سند:** تا تحویلِ اولین برج، هیچ Feature خارج از چرخهٔ اولین پروژه.

⏳ **جلد۵۸ (معماری جهانی World→Country→…)**: درختِ geo فعلی استان→شهر→منطقه→محله است؛ لایهٔ کشور/ارز/قانونِ چندکشوری → تریگر: ورودِ Region دوم. ⏳ **جلد۵۹ (Enterprise/Government/Platform Economy)**: خودِ سند هم آن را به بعدِ نسخهٔ ۱ موکول کرد — فقط ثبت.
⏳ **جلد۶۰ (Workflow خودِ بازی)**: MelkJet برای پنل‌ها workflow-store دارد؛ نسخهٔ گیم‌پلی (Trigger/Condition/Action روی رویدادهای بازی + Builder بصری) → صف، بعد از چرخهٔ اولین پروژه.
✅ **جلد۶۱ — فاز ۱۲**: شرکت+برند+اعتبار ستاره‌ای+استخدام (دفتر/تجهیزات → با جلد ۶۴). ✅ **جلد۶۲ هسته**: زمین از آگهی‌های واقعی + شخصیتِ مالک + مذاکره + توضیحِ AI (رنگ‌بندی نقشه/رقیبِ زنده → NPC/نقشه). ✅ **جلد۶۳ هسته**: پروانه (درخواست/عوارض/اعتراض/کمیسیون/صدور). ⏳ ورودِ سه‌بعدی شهرداری/بازرسِ متحرک → Block 8.
✅ **جلد۶۴–۷۲ (هستهٔ Engine) — فاز ۱۳**: چرخهٔ کاملِ ساخت+نقدینگی+پیش‌فروش+فروش (بالا). ⏳ باقی از این جلدها: انتخابِ برند/پیمانکارِ چندگزینه‌ای برای MEP (جلد ۶۷ جزئیات)، هویتِ نما/سبکِ معماری با Preview (جلد ۶۸ — بخشی qualityFactor پوشش می‌دهد؛ Preview بصری → Visual Bible/Pass 2)، Project Health مرکب، گزارشِ شبانهٔ کارگاه در نامهٔ روزانه، Customer Engine (شخصیتِ خریدارِ واحد — مثل ownerPersona)، Auction/Tender Engine (مناقصه/مزایده/مشارکت به‌جای خریدِ مستقیم — «شروعِ هر پروژه خودش بازی است»).
📌 **تصمیمِ سند: از این به بعد جلدها فقط Engine یا Core Loop** — سناریو/محتوا بعداً (Scenario Packs). Visual/Art Bible به‌عنوان Pass 2 جداگانه.

## 🔁 روالِ هر سند جدید (برای سشن‌های آینده)

1. سند را **کامل** بخوان (چند Read با offset).
2. همین فایل را به‌روز کن: جلد جدید + وضعیت + آنچه قابل‌ساختِ «واقعی» است.
3. فقط چیزهایی را بساز که با دادهٔ واقعی کار کنند؛ بقیه را با دلیل و Trigger در ⏳ ثبت کن.
4. تست یونیت+PG، tsc، build، کامیت (تریلر استاندارد)، push، و این فایل را هم در همان کامیت به‌روز کن.
