## 🧭 قانون اساسیِ طراحی (جلد ۷۷/۷۸ — تغییرناپذیر)

**Core Loop:** فرصت → تصمیم → اجرا → نتیجه → رشد → فرصتِ بعدی.
**Golden Rule:** هر عملِ بازیکن باید حداقل یکی از ۴ شاخص را تغییر دهد: پول / زمان / اعتبار / دارایی.
**۸ ستون (Pillars):** ۱-تصمیم نه عملِ فیزیکی ۲-هر انتخاب پیامد دارد ۳-اقتصادِ زنده ۴-ساختمان وسیلهٔ رشد است نه هدف ۵-بازیکن داستانش را می‌سازد ۶-واقعی اما لذت‌بخش ۷-پیشرفتِ همیشگی ۸-مقیاس‌پذیر تا ابد.
**دروازهٔ Feature (۴ سؤال):** تصمیمِ جدید می‌سازد؟ به کدام Loop وصل است؟ نتیجهٔ قابل‌مشاهده دارد؟ رشد/تغییر می‌دهد؟

**مکملِ GDD جدید (Chapter 04 — از این به بعد «فصل»، نه «جلد»):**
بازیکن **CEO** است، نه حسابدار/اپراتور؛ «اگر بازیکن از انجامِ کاری لذت نمی‌برد → AI انجامش دهد»؛
هر سیستم فقط **۳–۸ تصمیمِ مهم**؛ «قابلیتی که تصمیمِ جالب نمی‌سازد، اصلاً واردِ GDD نمی‌شود»؛
بازار باید به **رفتارِ بازیکن** واکنش نشان دهد (اشباعِ عرضه ✓ فاز ۱۵؛ اشباعِ لوکس/جنتریفیکیشنِ محله → NPC).
🚫 قابلیت‌های ERPوار (دفترِ کل/ترازنامه/مغایرتِ بانکی/حضورغیاب/انبارِ چندسطحی) به تصمیمِ خودِ سند ردشده‌اند — هرگز.

**پیوتِ «مدل B» (سند ۱۴ — حاکم بر همهٔ فازهای بعد):** MelkJet **بازی** است، نه شبیه‌ساز —
Tycoon + Strategy + Collection + Live Economy. آزمونِ هر قابلیت: «آیا کاربر می‌گوید *فقط ۵ دقیقهٔ دیگر*؟»
و هر قابلیت باید **تصمیم / ریسک / فرصتِ** جدید بسازد. ترتیبِ طراحی: **Hook → Daily Gameplay → Meta Progression → پشت‌صحنه.**
دنیا منتظرِ بازیکن نمی‌ماند (فرصت‌های محدودِ روزانه ✓ فاز ۱۶)؛ هر شخصیت **حافظه** دارد (حافظهٔ مذاکره ✓ فاز ۱۶)؛
اعتبار فقط عدد نیست — روی مذاکره/بانک/فرصت‌ها اثرِ واقعی دارد (✓ فاز ۱۶). Big Data محصولِ طبیعیِ رفتار است، نه حسِ بازیکن.

**بستنِ Core Gameplay (سند ۱۵ — فصل ۴ تمام شد):** متا-لوپ: کشف → تصمیم → سود/زیان → رشد → فرصتِ جدید.
**۷ ستونِ نهایی:** کشفِ فرصت · معامله · ریسک · رقابت · رشد · جمع‌آوریِ دارایی · رویدادهای زنده — قابلیتی که به
هیچ‌کدام کمک نکند، وارد نمی‌شود. طراحی **Opportunity-محور** است نه Task-محور. Progression یعنی «امکاناتِ بیشتر،
نه اعدادِ بزرگ‌تر» (سطح‌گشاییِ بازارِ سرمایه/شرکت/مشارکت + ظرفیتِ پروژهٔ همزمان ✓ فاز ۱۷).
**قانونِ بصری (سند ۲۰ — فصل ۱۰):** معناشناسیِ ثابتِ رنگ — سبز(#7c6)=سود/موفق · قرمز(#e88)=ضرر/خطر ·
نارنجی(#e7a14a)=فرصت/هشدار · طلایی(--gold)=ویژه/برند؛ هر رنگ فقط یک مفهوم. **هیچ انیمیشنِ اجباریِ
بالای ۱.۵ ثانیه**؛ احترام به reduced-motion؛ مراحلِ ساخت از ظاهر قابلِ تشخیص (استپر + برجِ 🏗 در خطِ آسمان).

**قانونِ UI (سند ۱۹ — فصل ۹، برای هر صفحه/کارت):** هر صفحه فقط به ۴ سؤال جواب می‌دهد —
۱) الان وضعیت چیست؟ ۲) چه فرصت/مشکلی هست؟ ۳) بهترین اقدامِ بعدی چیست؟ ۴) با کدام دکمه؟
واژهٔ رسمی: **پرتفوی** (نه Inventory). مهم‌ترین اقدامِ بعدی بالای صفحه (mentorLine)؛ وضعِ بحرانی با رنگِ هشدار.

**قوانینِ اقتصاد (فصل ۵):** پول ابزار است نه هدف؛ نقدینگی مهم‌تر از ثروت؛ هر تصمیم هزینهٔ فرصت دارد؛ هیچ درآمدی
تضمینی نیست؛ ارزِ قابلِ‌خرج حداکثر ۲ (تومانِ شبیه‌سازی + ملک‌کوین)؛ XP/اعتبار غیرقابلِ‌خرید؛ اعداد → Game Balance.
